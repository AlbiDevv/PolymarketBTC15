# PolymarketBTC15

Исследовательский Python-проект для анализа маркетов Polymarket, обработки рыночных данных и симуляции maker-логики в paper/shadow-режиме.

Проект собран как инженерный стенд: получение и нормализация данных, риск-логика, работа с API, мониторинг, алерты и локальный dashboard. Основной акцент — на архитектуре, обработке данных и безопасной симуляции торговых сценариев.

## Что демонстрирует проект

- интеграцию с внешними API и CLOB-интерфейсом;
- проектирование модулей для market data, execution, risk и monitoring;
- работу с асинхронными задачами и сервисной архитектурой;
- безопасное хранение чувствительных параметров через переменные окружения;
- разделение исследовательского режима, shadow-режима и боевого исполнения;
- подготовку инфраструктурной документации для локального и серверного запуска.

## Основные возможности

- клиент для работы с Polymarket/CLOB;
- модуль кошелька для Polygon с загрузкой ключа только из окружения;
- риск-менеджмент и логика принятия решений;
- мониторинг состояния системы;
- Telegram-алерты и аналитические уведомления;
- локальный dashboard;
- deployment-документация для shadow-режима.

## Стек

- **Python**
- **asyncio**
- **Polymarket / CLOB API**
- **Polygon / Web3**
- **Telegram Bot API**
- **Docker**

## Безопасность

- реальные секреты не должны храниться в коде и git;
- приватный ключ загружается только через `POLYGON_PRIVATE_KEY`;
- `.env.example` содержит только шаблоны переменных;
- shadow/paper-режим используется как безопасный режим по умолчанию;
- параметры API и кошелька задаются через переменные окружения.

## Переменные окружения

Основные переменные описаны в `.env.example`:

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `TELEGRAM_ADMIN_CHAT_ID`
- `TELEGRAM_ADMIN_CHAT_IDS`
- `POLYGON_PRIVATE_KEY`
- `POLYMARKET_API_KEY`
- `POLYMARKET_API_SECRET`
- `POLYMARKET_API_PASSPHRASE`
- `QWEN_API_KEY`

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Создайте `.env` на основе `.env.example` и заполните только те значения, которые нужны для выбранного режима запуска.

## Важно

Проект предназначен для исследования market data, тестирования risk-логики и paper/shadow-сценариев. Это не финансовая рекомендация и не инструкция по торговле.
